1.DOM基础

文档对象模型( Document Object Model,简称DOM),是W3C组织推荐的处理可扩展标记语言(HTML
或者XML)的标准编程接口

W3C已经定义了一系列的DOM接口,通过这些DOM接口可以改变网页的内容、结构和样式

  • 获取元素
    • getElementById获取元素
      • 返回的是对象
      • 获取的是该Id里的内容
    • getElementsByTagName()返回带有指定标签名的对象的集合
      • 以伪数组的形式存储
      • 上面两个可以一起使用,即如下所示:
    • 通过HTML5新方法
      • getElementsByClassName()
      • querySelector(‘选择器’) 根据指定选择器返回第一个元素对象
        • 可用于选择类 ID 元素
        • 缺陷是只能选中第一个选择器
      • querySelectAll(‘选择器’) 根据选择器返回所有
    • 获取body元素
      • 使用document.body
    • 获取html元素
      • 使用document.documentElement

  • 事件基础

    • 事件概述
      • 网页中的每个元素都可以产生某些可以触发 Javascript的事件,例如,我们可以在用户点击某按钮时产生一个事件,然后去执行某些操作
    • 事件由三部分组成:事件源/事件类型/事件处理程序
      • 事件源:事件被触发的对象 谁被触发
      • 事件类型:如何触发 什么事件触发的 比如鼠标点击等
      • 事件处理程序:即出发后怎么响应 通过一个函数赋值的方法完成
    • 执行事件的步骤
      • 获取事件源
      • 注册事件(绑定事件)
      • 添加事件处理程序(采取函数赋值形式)
      • 常见的鼠标事件
    • 操作元素
      • 改变元素内容
        • element.innerText:从起始位置到终止位置的内容,但它去除html标签,同时空格和换行也会去掉
        • element.innerHTML:起始位置到终止位置的全部内容,包括html标签,同时保留空格和换行
      • 表格元素的更改
        • 如input,需要更改的是其value值
      • 样式属性操作
        • element.style 行内样式操作 (修改元素较少)
        • element.className 类名样式操作
          • 但是此类方式会改变原元素的类名,所以需要在更改前将原类名也添加上
      • 得到元素的属性
        • element.属性
          • 获取的是内置属性,元素本身自带的属性
        • element.getAttribute(‘属性’)
          • 主要获得自定义的属性
      • 改变元素的属性值
        • 使用’=’
        • 使用setAttribute()
      • 属性的移除
        • removeAttribute()
      • H5自定义属性
        • 使用自定义属性data开头,作为自定义属性,如data-index
        • 获取方式为:
        • dataset方式是H5新增的,同时只能获取data开头的自定义属性
    • 节点操作
      • 一般地,节点至少拥有 node Type(节点类型)、 nodename(节点名称)和 nodevalue(节点值)这三个
        基本属性
        • 元素节点 nodeType 为 1
        • 属性节点 nodeType 为 2
        • 文本节点 nodeType 为 3 (文本节点包括文字/空格/换行等)
        • 实际开发中,节点操作主要操作的是元素节点
      • 节点层级
        • 子节点:parentNode.childNodes 返回包含指定节点的子节点的集合,该集合为及时更新的集合
        • 使用parentNode.children 返回子元素节点,不会返回其他杂乱节点
        • firstChild/lastChild 获取第一个和最后一个节点(不论是文本节点还是元素节点)
        • firstElementChild/lastElementChild 获取的是元素节点 但是有兼容性问题
        • 实际开发中取得固定元素节点,使用的是children[i]
        • 兄弟节点 nextSibling/previousSibling 上一个/下一个兄弟节点
        • 同理 nextElementSibling是只返回元素节点 同样有兼容性问题
      • 创建节点
        • document. createELement()方法创建由 tagname指定的HTML元素。因为这些元素原先不存在,是根据我们的需求动态生成的,所以我们也称为动态创建元素节点
      • 添加节点
        • node. appendchild()方法将一个节点添加到指定父节点的子节点列表末尾。类似于css里面的
          after伪元素
        • node. insertBefore()方法将一个节点添加到父节点的指定子节点前面。类似于css里面的 before
          伪元素。
      • 删除节点
        • node.removeChild(child)
      • 复制节点
        • node.cloneNode() 复制节点
        • 如果括号参数为空或者为 false,则是浅拷贝,即只克隆复制节点本身,不克隆里面的子节点及内容
        • 深拷贝的话 只需要将括号内添加true即可

2.事件高级

  • 注册事件

    • 概述:给元素添加事件,称为注册事件或者绑定事件

      • 注册事件分为两种:传统方式和方法监听注册方式

      • 传统注册方式:

        • 利用on开头的事件,如onclick
        • 特点:注册事件的唯一性
        • 即同个元素同个事件只能设置一个处理函数,最后注册的处理函数将会覆盖前面注册的处理函数

      • 方法监听注册方式

        • w3c标准 推荐方式
        • addEventListener()它是一个方法
        • IE9之前的IE不支持此方法,可使用addEvent(0代替
        • 特点:同一个元素可以注册多个监听器
        • 按注册顺序依次执行

      • addEventListener 事件监听方式

      • eventTarget.addEventlistener(type,listener[,useCapture])
        • 该方法将指定的监听器注册到eventTarget目标对象上,当对该对象触发指定的事件时,就会执行事件处理函数.
        • type:事件类型字符串,比如click/mouseover
        • listener:事件处理函数,事件发生时,会调用该监听函数
        • useCapture:可选参数,是一个布尔值,默认是false

  • 删除事件(解绑事件)

    • 删除事件的方式
      • 传统方式:元素.onclick = ‘null’;即可
      • 方法监听注册方式:元素.removeEventListener(type,函数名) 注意:要想删除事件,必须在注册事件时将绑定的函数独立出来

  • DOM事件流

    • 事件流描述的是从页面中接收事件的顺序
    • 事件发生时会在元素节点之间按照特定的顺序传播,这个传播过程即DOM事件流。
    • DOM事件流分为三个阶段:
      • 捕获阶段
      • 当前目标阶段
      • 冒泡阶段
    • addEventlistener既能执行捕获也能执行冒泡 默认是冒泡,想执行捕获需要将第三个参数改为true
      • onclick只能执行冒泡,即从底层开始,向上依次触发
      • 捕获阶段从上向下依次触发

  • 事件对象

    • 事件对象是我们事件的一系列相关数据的集合,跟事件相关的,比如鼠标点击里面就包含了鼠标的相关信息
    • 使用时只需在function()中添加e即可,即function(e)
      • 注意:如果需要知道是谁触发了事件 使用e.target获取该对象
      • this返回的是绑定事件的对象 如设置ul的事件,但是点击的是里面li触发
    • 阻止默认事件 如让链接不跳转
      • function()函数里添加一行代码:e.preventDefault()即可

  • 阻止事件冒泡

    • 阻止冒泡的两种方式:
      • 在函数里面添加:e.stopPropagation(); 即可停止向上冒泡
      • 非标准使用e.cancelBubble = true;

  • 事件委托

    • 原理:不是每个子节点单独设置事件监听器,而是事件监听器设置在其父节点上,然后利用百泡原理影响设置每个子节点

  • 常用的鼠标事件

    • 阻止选中:selectstart

  • 常用的键盘事件

    • 键盘事件对象
      • keyCode:返回键盘事件对象的ASCII值
      • 此外,onkeydown和onkeyup不区分字母大小写,onkeypress区分字母大小写
      • keypress不识别功能键,但是keyCode属性能区分大小写,饭后不同的ASCII值
