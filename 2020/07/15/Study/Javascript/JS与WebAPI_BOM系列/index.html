一.BOM概述

    1. 1 什么是BOM

    BOM(Browser Object Model) 即浏览器对象模型,它提供了独立于内容而与浏览器窗口进行交互的对象,其核心
    对象是 windows

    BOM由一系列相关的对象构成,并且每个对象都提供了许多方法和属性

  • ​ 1.2 BOM的构成

    ​

    ​ windows对象是window对象是浏览器的顶级对象,它具有双重角色
    1.它是JS访问浏览器窗口的一个接口
    2.它是一个全局对象。定义在全局作用域中的变量、函数都会变成 window对象的属性和方法。

二.window对象的常见事件

  • 2.1 窗口加载事件

    • window. onload是窗口(页面)加载事件,当文档内容完全加载完成会角触发该事件(包括图像、脚本文件、CSS文件等),就调用的处理函数
    • 有了该加载事件,我们可以将JS代码写到页面元素上方,且如果有多个事件,只会加载最后一个

  • 2.1 调整窗口大小事件

    • window.onresize是调整窗口大小加载事件,当触发时就调用的处理函数

    • 只要窗口大小发生像素变化,就会触发这个事件

    • 利用这个事件完成响应式布局,window.innerWidth是当前屏幕的宽度

三.定时器

  • 3.1两种定时器
    • window对象给我们提供了两个非常好用的定时器
    • setTimeout()
      • setTimeout( 执行函数, 延迟时间/毫秒)
      • 在调用时可以直接使用,无前缀
      • 延时时间可以省略,默认为0
    • setInterval(执行函数, 调用间隔时间/毫秒)
      • 多次调用里面的函数
  • 3.2 停止定时器
    • 使用clearTimeout(定时器名字)
    • 使用clearInterval(定时器名字)
  • 3.3 this
    • this的指向在函数定义的时候是确定不了的,只有函数执行的时候才能确定this到底指向谁,一般情兄下this
      的最终指向的是那个调用它的对象

四.JS执行机制

  • 分为:同步和异步
    • JS的异步是通过回调函数实现的
    • 执行过程:
        1. 顺序执行代码 遇到同步任务依次执行 遇到异步任务放入新的任务队列
        2. 依次将异步任务放入任务队列
        3. 执行完所有的同步任务,返回执行异步任务
      • :man_facepalming: :comet:

五.Location对象

  • 5.1 location对象概述
    • window对象给我们提供了一个 location属性用于获取或设置窗体的URL,并且可以用于解析URL。因为
      这个属性返回的是一个对象,所以我们将这个属性也称为 location对象,
    • 锚点:fragment
  • 5.2 location对象属性

六.navigator对象

  • navigator对象包含有关浏览器的信息,它有很多属性,我们最常用的是 userAgent,该属性可以返回由客
    户机发送服务器的ue- agent头部的值

七.history对象

  • 三个对象方法(记录浏览器历史)
    • forward:前进
    • back:后退
    • go:下一步
